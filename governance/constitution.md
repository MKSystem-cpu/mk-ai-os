# MK AI OS Constitution
1. One module, one clear responsibility.
2. Use the fewest necessary skills.
3. Core skills remain brand-agnostic.
4. Brand rules are injected from Brand Modules.
5. Workflows define order; skills perform work; Manager decides routing.
6. Do not ask again for information already provided.
7. Begin immediately when inputs are sufficient.
8. Quality control is integrated from the first step, not added at the end.
9. Avoid duplicate stages and duplicate instructions.
10. Every module uses semantic versioning and a changelog.
